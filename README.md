# ClarityMD

ClarityMD is a three-panel web application for orthopedic procedure decision support. A surgeon enters a patient profile, selects the affected joint on an interactive body diagram, the backend ML engine ranks matching Arthrex procedures, and the app returns:

- A ranked procedure list with clinical notes, risk flags, and confidence scores
- A clinical surgeon brief
- A plain-language patient education summary

AI summaries are generated by Groq (Llama 3.3 70B) on the backend.

## Current Architecture

- Frontend React app handles intake via a patient form and an interactive SVG body selector
- Backend Flask API performs TF-IDF ML ranking and Groq AI summary generation
- Canonical procedure catalog: `backend/data/procedures.json` (74 procedures across 11 joint areas)
- Backend auto-seeds SQLite from catalog on startup; new entries are added incrementally

## Functional Coverage (Mini SRS)

- FR1 Patient Profile Input: age, sex, joint/area (selected via body diagram or dropdown), diagnosis/symptoms, activity level, prior treatments, required field validation
- FR2 ML Recommendation Engine: TF-IDF ranked procedures with relevance score and confidence labels (Strong / Good / Possible Match)
- FR3 Surgeon View: top ranked procedures with product, category, technique notes, recovery timeline, contraindications, and profile risk flags
- FR4 Patient View: plain-language summary, recovery timeline, and AI disclaimer
- FR5 Groq-powered dual summaries: clinical brief for the surgeon and conversational summary for the patient, generated via Groq Llama 3.3 70B

## Quick Start

### 1) Backend

```bash
cd backend
python -m venv venv
# Windows
venv\Scripts\activate
# macOS/Linux
# source venv/bin/activate

pip install -r requirements.txt
python app.py
```

Create `backend/.env` with:

```
GROQ_API_KEY=your_key_here
```

### 2) Frontend

```bash
cd frontend
npm install
cp .env.example .env
npm start
```

Frontend default backend URL:

```
REACT_APP_BACKEND_URL=http://localhost:5000
```

## API

### `POST /recommend`

Required payload fields:

- `age`
- `sex`
- `joint`
- `diagnosis`
- `activity`
- `prior_treatments`

Optional:

- `pain_types` (auto-inferred when omitted)

Response includes:

- `procedures[]` (ranked with `relevance_score`, `confidence_label`, `contraindication_flags`)
- `surgeonSummary`
- `patientSummary`

### `GET /health`

Returns service status and seeded procedure count.
