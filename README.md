# ClarityMD

ClarityMD is a dual-view web app for orthopedic procedure decision support. A surgeon enters a patient profile, the backend ML engine ranks Arthrex procedures, and the app returns:

- A clinical surgeon brief
- A plain-language patient education summary

AI summaries are generated by Groq (Llama 3.3 70B) on the backend.

## Current Architecture

- Frontend React app handles intake and display
- Backend Flask API performs ML ranking and AI summary generation
- Canonical procedure catalog: `backend/data/procedures.json`
- Backend auto-seeds SQLite from catalog on first run

## Functional Coverage (Mini SRS)

- FR1 Patient Profile Input: age, sex, joint/area, diagnosis/symptoms, activity level, prior treatments, required field validation
- FR2 ML Recommendation Engine: ranked procedures with relevance score and confidence labels
- FR3 Surgeon View: top recommendations, product/category/technique/recovery, contraindications and profile risk flags
- FR4 Patient View: plain-language summary, expectations, and recovery outlook
- FR5 Claude-style dual summaries: implemented via Groq backend prompts (clinical vs conversational tone)
- FR6 ElevenLabs Voice: not implemented (stretch)

## Quick Start

### 1) Backend

```bash
cd backend
python -m venv venv
# Windows
venv\Scripts\activate
# macOS/Linux
# source venv/bin/activate

pip install -r requirements.txt
python app.py
```

Create `backend/.env` with:

```
GROQ_API_KEY=your_key_here
```

### 2) Frontend

```bash
cd frontend
npm install
cp .env.example .env
npm start
```

Frontend default backend URL:

```
REACT_APP_BACKEND_URL=http://localhost:5000
```

## API

### `POST /recommend`

Required payload fields:

- `age`
- `sex`
- `joint`
- `diagnosis`
- `activity`
- `prior_treatments`

Optional:

- `pain_types` (auto-inferred when omitted)

Response includes:

- `procedures[]` (ranked with `relevance_score`, `confidence_label`, `contraindication_flags`)
- `surgeonSummary`
- `patientSummary`

### `GET /health`

Returns service status and seeded procedure count.
